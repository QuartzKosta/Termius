## Поля записи в админке

При создании/редактировании записи в админке (`/admin`):

```
NAME *           — название записи (обязательно)
CATEGORY         — категория (UNDEAD, FIEND, LORE, и т.д.)
TITLE            — подзаголовок/эпитет
DESCRIPTION      — основной текст. Поддерживает [[HIDDEN:текст]] для скрытых фрагментов
IMAGE URL        — ссылка на изображение (jpg/png/gif/webp)
SIGIL            — сигила (i-skull, i-eye, i-serpent, i-crown, i-flame, i-hourglass, i-ritual, i-god)
IS LOCKED        — запечатана ли запись (boolean)
PUZZLE TYPE      — тип головоломки (none/keyword/tumbler/constellation/circuit/alchemy/runes/sliding/g2048/memory/maze/fragment/meta)
PUZZLE HINT      — текстовая подсказка для игрока
PUZZLE DATA      — JSON с параметрами головоломки (формат зависит от типа)
SHARD WORD       — слово-осколок для meta-головоломки
```

### Логика работы

```
Запись is_locked=true + puzzle_type="keyword"
  → Игрок видит карточку [ДАННЫЕ ПОВРЕЖДЕНЫ]
  → Кнопка "⚠ СНЯТЬ ПЕЧАТЬ"
  → Открывается bypass overlay с головоломкой
  → При успехе: is_locked → false (через /api/archive/unlock)
  → Запись становится читаемой

Запись is_locked=false + puzzle_type="keyword" + description содержит [[HIDDEN:...]]
  → Игрок видит открытую карточку
  → Часть описания censored + кнопка "🔓 РАСШИФРОВАТЬ"
  → Открывается bypass overlay (title: "РАСШИФРОВКА ФРАГМЕНТА")
  → При успехе: фрагмент раскрывается (без изменения is_locked)
```

## 1. Keyword (кодовое слово)

Игрок должен ввести правильное слово. Регистр не учитывается.

### Puzzle Data

```json
{
  "answer": "пепел"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `answer` | string | Правильный ответ (приводится к lowercase, trim) |

### Пример

**PUZZLE TYPE:** `keyword`

**PUZZLE HINT:** `То, что остаётся от клятвы, когда угасает последняя искра. Назови это слово.`

**PUZZLE DATA:**
```json
{"answer": "пепел"}
```

**Логика:**
1. Игрок видит подсказку и поле ввода
2. Вводит слово
3. Нажимает «ПОДТВЕРДИТЬ» или Enter
4. Если `answer.toLowerCase().trim() === input.toLowerCase().trim()` → успех
5. Иначе → `bypassFail("неверное слово")` + cooldown

## 2. Tumbler (барабанный замок)

Три барабана с символами. Игрок вращает ▲/▼ чтобы выставить правильную последовательность.

### Puzzle Data

```json
{
  "symbols": ["ᚠ", "ᚢ", "ᚦ", "ᚨ", "ᚱ", "ᚲ"],
  "correct": ["ᚨ", "ᚱ", "ᚦ"]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `symbols` | string[] | Массив доступных символов на барабанах |
| `correct` | string[] | Правильная последовательность (3 символа) |

### Пример

**PUZZLE TYPE:** `tumbler`

**PUZZLE HINT:** `Три руны встанут в ряд: первая — бог, вторая — поездка, третья — терние. Вращай барабаны.`

**PUZZLE DATA:**
```json
{
  "symbols": ["ᚠ", "ᚢ", "ᚦ", "ᚨ", "ᚱ", "ᚲ"],
  "correct": ["ᚨ", "ᚱ", "ᚦ"]
}
```

**Логика:**
1. 3 барабана, каждый показывает случайный символ (не совпадающий с correct)
2. ▲/▼ кнопки вращают барабан (+1/-1 по массиву symbols)
3. При совпадении символа с correct[i] — рамка барабана зелёнеет
4. Когда все 3 совпадают → успех (автоматически)

## 3. Constellation (соединение звёзд)

На SVG-поле расположены звёзды. Игрок должен кликнуть их в правильном порядке.

### Puzzle Data

```json
{
  "nodes": [
    {"x": 15, "y": 25, "id": "eye"},
    {"x": 45, "y": 15, "id": "crown"},
    {"x": 75, "y": 35, "id": "flame"},
    {"x": 35, "y": 70, "id": "serpent"},
    {"x": 65, "y": 80, "id": "tail"}
  ],
  "order": ["eye", "crown", "flame"]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `nodes` | array | Звёзды с координатами (x: 0-300, y: 0-280) и id |
| `order` | string[] | Правильный порядок кликов по id |

### Пример

**PUZZLE TYPE:** `constellation`

**PUZZLE HINT:** `Соедини Око, Корону и Пламя — в этом порядке. Звёзды укажут путь.`

**PUZZLE DATA:**
```json
{
  "nodes": [
    {"x": 15, "y": 25, "id": "eye"},
    {"x": 45, "y": 15, "id": "crown"},
    {"x": 75, "y": 35, "id": "flame"},
    {"x": 35, "y": 70, "id": "serpent"},
    {"x": 65, "y": 80, "id": "tail"}
  ],
  "order": ["eye", "crown", "flame"]
}
```

**Логика:**
1. Все звёзды из `nodes` отображаются (включая "лишние")
2. Игрок кликает звёзды
3. Если клик правильный (совпадает с `order[currentStep]`) → звезда загорается (amber)
4. Если неправильный → `bypassFail("неверный порядок звёзд")`, прогресс сбрасывается
5. Когда все звёзды в `order` кликнуты → успех

## 4. Circuit (трубы маны)

Сетка 3×3 с L/I/T-образными трубами. Нужно соединить источник (●cyan) со стоком (●amber).

### Puzzle Data

```json
{
  "solution": [0, 1, 2, 5, 8]
}
```

> **Важно:** Поле `solution` сейчас **не используется** для проверки. Код генерирует случайные типы труб и позиции source/drain при каждом открытии. Проверка идёт через BFS (реальная связность труб).

| Поле | Тип | Описание |
|---|---|---|
| `solution` | number[] | (необязательно) резервное поле |

### Пример

**PUZZLE TYPE:** `circuit`

**PUZZLE HINT:** `Поверните трубы от источника (●) к стоку (●).`

**PUZZLE DATA:**
```json
{"solution": [0, 1, 2, 5, 8]}
```

**Логика:**
1. 9 ячеек (3×3), каждая со случайным типом трубы: L (50%), I (30%), T (20%)
2. Source и drain — случайные позиции на краях сетки
3. Клик по ячейке вращает трубу на 90°
4. BFS проверяет: есть ли непрерывный путь от source к drain
5. Когда путь есть → success, ячейки на пути подсвечиваются зелёным

## 5. Alchemy (алхимический рецепт)

Игрок должен заполнить слоты ингредиентами в правильном порядке.

### Puzzle Data

```json
{
  "ingredients": ["🜂", "🜄", "🜁", "🜃", "☿"],
  "correct_recipe": ["🜃", "🜄", "☿"]
}
```

| Поле | Тип | Описание |
|---|---|---|
| `ingredients` | string[] | Доступные ингредиенты (эмодзи) |
| `correct_recipe` | string[] | Правильная последовательность |

### Пример

**PUZZLE TYPE:** `alchemy`

**PUZZLE HINT:** `Земля — основа, Вода — проводник, Ртуть — связующее. Три капли в этом порядке.`

**PUZZLE DATA:**
```json
{
  "ingredients": ["🜂", "🜄", "🜁", "🜃", "☿"],
  "correct_recipe": ["🜃", "🜄", "☿"]
}
```

**Логика:**
1. Показываются слоты (количество = `correct_recipe.length`)
2. Под ними — ингредиенты из `ingredients`
3. Клик по ингредиенту → заполняет следующий слот
4. Использованные ингредиенты становятся недоступными
5. Когда все слоты заполнены → проверка
6. Если последовательность совпадает → успех
7. Иначе → `bypassFail("неверная рецептура")`

## 6. Runes (руническая последовательность)

Показана последовательность рун с пропуском в конце. Игрок выбирает правильную руну из вариантов.

### Puzzle Data

```json
{
  "sequence": ["🜂", "🜃", "🜁", "🜄"],
  "options": ["🜂", "ᚦ", "🜃", "ᚱ"],
  "correct": "🜃"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `sequence` | string[] | Последовательность рун (последняя — "?") |
| `options` | string[] | Варианты ответа |
| `correct` | string | Правильная руна |

### Пример

**PUZZLE TYPE:** `runes`

**PUZZLE HINT:** `Стихии: Огонь, Земля, Воздух, Вода... Какая завершит пятый шаг?`

**PUZZLE DATA:**
```json
{
  "sequence": ["🜂", "🜃", "🜁", "🜄"],
  "options": ["🜂", "ᚦ", "🜃", "ᚱ"],
  "correct": "🜃"
}
```

**Логика:**
1. Показывается sequence + "?" слот
2. Под ним — варианты из `options`
3. Клик по варианту → проверка
4. Если `r === correct` → успех (через 400мс)
5. Иначе → `bypassFail("неверная руна")`, вариант краснеет

## 7. Fragment (осколок памяти)

Запись содержит осколок истинного имени бога. Осколок **собирается автоматически** при прочтении записи.

### Puzzle Data

Не требуется. `puzzle_type = "fragment"` без `puzzle_data`.

### Дополнительно

Поле **SHARD WORD** — слово-осколок (например `ПЕПЕЛ`, `ПОМНИТ`, `ИМЯ`).

### Пример

**PUZZLE TYPE:** `fragment`

**SHARD WORD:** `ПЕПЕЛ`

**PUZZLE HINT:** `Прочтите эту запись для осколка.`

**Логика:**
1. Запись должна быть **открытой** (`is_locked = false`)
2. При клике на запись → `tryCollectShard(record)` вызывается автоматически
3. Осколок добавляется в `STORE.collectedFragments`
4. Sidebar показывает собранные осколки
5. Выдаётся достижение `SHARD_COLLECTOR` (первый осколок)
6. Нужно собрать **все 5 осколков** для meta-головоломки

## 8. Meta (финальная фраза)

Финальная головоломка. Требует **все осколки** собранными. Игрок должен составить из них фразу.

### Puzzle Data

```json
{
  "answer": "ПЕПЕЛ ПОМНИТ ИМЯ БОГА ОСВОБОДИ"
}
```

| Поле | Тип | Описание |
|---|---|---|
| `answer` | string | Правильная фраза (приводится к uppercase, trim) |

### Пример

**PUZZLE TYPE:** `meta`

**PUZZLE HINT:** `Соберите пять слов из пяти записей-фрагментов. Сложите их в фразу.`

**PUZZLE DATA:**
```json
{"answer": "ПЕПЕЛ ПОМНИТ ИМЯ БОГА ОСВОБОДИ"}
```

**Логика:**
1. Показываются все собранные осколки (из `STORE.collectedFragments`)
2. Если собраны не все → «найдите все записи-осколки»
3. Если все собраны → поле ввода + кнопка «ПРОИЗНЕСТИ»
4. Проверка: `input.toUpperCase().trim() === answer.toUpperCase().trim()`
5. При успехе → `unlockAchievement("FINAL_REVELATION")` + `bypassSuccess()`

### Осколки для meta

5 записей с `puzzle_type = "fragment"` + `shard_word`:

| Осколок | Где |
|---|---|
| `ПЕПЕЛ` | The Fragment-Bearer (NPC) |
| `ПОМНИТ` | The Shard of Memory (Lore) |
| `ИМЯ` | The Fragment-Bearing Queen (Ruler) |
| `БОГА` | The Hour-Keeper (NPC) |
| `ОСВОБОДИ` | The Cinder-King (Ruler) |

## Hidden Fragments (скрытые фрагменты)

**Механика:** часть описания открытой записи закрыта отдельной мини-головоломкой.

### Синтаксис

В поле **DESCRIPTION** вставьте маркер:

```
Видимый текст записи. [[HIDDEN:Секретный текст, который раскроется после решения головоломки.]] Продолжение видимого текста.
```

### Условия

- Запись должна быть **открытой** (`is_locked = false`)
- Запись должна иметь `puzzle_type` (не `none`) + `puzzle_data`
- `puzzle_type` и `puzzle_data` **переиспользуются** — та же головоломка, что и для снятия печати

### Пример

**IS LOCKED:** `false` (запись открыта)

**DESCRIPTION:**
```
О'Кол Ливенант — один из наиболее заметных представителей рода Ливенантов. Он управляет пятью городами. [[HIDDEN:Истинное имя О'Кола — Азраэль Третий, и он носит печать Хаоса на левом глазу.]]
```

**PUZZLE TYPE:** `keyword`

**PUZZLE DATA:**
```json
{"answer": "азраэль"}
```

**PUZZLE HINT:** `Его истинное имя скрыто. Назови имя — и фрагмент раскроется.`

### Что видит игрок

1. Описание: «О'Кол Ливенант — один из наиболее... Он управляет пятью городами.»
2. Дальше — **censored блок** (5 типов цензуры: чёрные блоки, glitch, strike, noise, blur)
3. Кнопка **«🔓 РАСШИФРОВАТЬ»**
4. Клик → bypass overlay с title «РАСШИФРОВКА ФРАГМЕНТА»
5. Решение → фрагмент становится зелёным, текст читаем
6. В **проекторе** фрагмент тоже censored → revealed

### Сохранение

Решённые фрагменты сохраняются в `localStorage` (`ashen_revealed_fragments`).

## Cooldown система

При ошибке в головоломке:

| Ошибка # | Cooldown | Дополнительно |
|---|---|---|
| 1-я | 3 секунды | Блокировка ввода |
| 2-я | 5 секунд | + Взгляд Бога +5% |
| 3-я+ | 10 секунд | Заморозка (box.frozen) |

Во время cooldown:
- Input и кнопки **disabled**
- Overlay с обратным отсчётом
- Puzzle заморожен (`pointer-events: none`)

## Анимация успешного решения

4 фазы (для полных unlock, не для fragment):

| Фаза | Время | Эффект |
|---|---|---|
| **CHARGING** | 0–0.8s | Box glow усиливается, progress bar 0→100%, «⟡ ЗАРЯДКА МАНЫ...» |
| **BURST** | 0.8–1.4s | Scale 1.12, 24 частицы, flash, «⚡ ПРОРЫВ!» |
| **REVEAL** | 1.4–1.8s | «✓ ДОСТУП РАЗРЕШЕН», server unlock, achievements |
| **UNVEIL** | 1.8s+ | showUnveil overlay: garbled→name анимация |

Для fragment puzzle: мгновенный успех (без 4-фазной анимации), простой success chord.

## 9. Sliding (пятнашки)

Классические пятнашки — расставить плитки 1-15 по порядку.

### Puzzle Data

```json
{
  "size": 4
}
```

| Поле | Тип | Описание |
|---|---|---|
| `size` | number | Размер сетки (4 = 4×4, 15 плиток + 1 пустая) |

### Логика

1. Перемешивание с проверкой решаемости (inversion count)
2. Клик по соседней с пустой плитке — сдвиг
3. Счётчик ходов
4. Авто-успех когда плитки 1,2,3...15 по порядку

## 10. G2048 (2048 с настраиваемой целью)

Объединяйте одинаковые числа, чтобы достичь заданной цели (не обязательно 2048).

### Puzzle Data

```json
{
  "target": 128,
  "size": 4
}
```

| Поле | Тип | Описание |
|---|---|---|
| `target` | number | Целевое число (128, 256, 512...) |
| `size` | number | Размер сетки (4 = 4×4) |

### Логика

1. Управление: стрелки ← ↑ → ↓, кнопки ▲◄▼►, свайп (mobile)
2. Счёт очков
3. Game over: нет ходов → bypassFail
4. Успех: достигли target → bypassSuccess

## 11. Memory (Саймон)

Запомните и повторите последовательность нажатий. Сложность растёт каждый раунд.

### Puzzle Data

```json
{
  "startLength": 3,
  "maxLength": 7
}
```

| Поле | Тип | Описание |
|---|---|---|
| `startLength` | number | Длина последовательности в первом раунде |
| `maxLength` | number | Максимальная длина (при достижении — успех) |

### Логика

1. 4 цветных pad (green/amber/red/cyan), каждый со своим тоном
2. Нажать СТАРТ → последовательность проигрывается (500ms на pad)
3. Игрок повторяет кликами
4. Раунд 1: 3 шага → Раунд 2: 4 шага → ... → Раунд N: maxLength шагов
5. Ошибка → bypassFail + cooldown, потом текущий раунд повторяется
6. Все раунды пройдены → bypassSuccess

## 12. Maze (Лабиринт во тьме)

Найдите выход в лабиринте с ограниченной видимостью. Врезались в стену — провал.

### Puzzle Data

```json
{
  "size": 8,
  "visionRadius": 2
}
```

| Поле | Тип | Описание |
|---|---|---|
| `size` | number | Размер сетки (8 = 8×8) |
| `visionRadius` | number | Радиус видимости вокруг игрока (Манхэттенское расстояние) |

### Логика

1. Лабиринт генерируется случайно (DFS algorithm) — каждый раз уникальный
2. Игрок видит только клетки в радиусе `visionRadius` (видимые)
3. Клетки на границе радиуса — полумрак (dim)
4. Остальные — тьма (невидимы)
5. Управление: стрелки ← ↑ → ↓, кнопки ▲◄▼►, свайп (mobile)
6. **Стена = провал!** → bypassFail + cooldown, потом лабиринт НЕ меняется
7. Достигли выхода (amber) → bypassSuccess
8. Счётчик ходов

### Пример

**PUZZLE TYPE:** `maze`

**PUZZLE HINT:** `Найдите выход в темноте. Стена = провал.`

**PUZZLE DATA:**
```json
{"size": 8, "visionRadius": 2}
```
